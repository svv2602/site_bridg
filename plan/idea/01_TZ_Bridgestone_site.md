# Технічне завдання на розробку сайту Bridgestone Україна

## 1. Загальна інформація про проєкт

**Назва проєкту:** Офіційний сайт шин Bridgestone для кінцевих споживачів в Україні.  
**Референс:** структура та UX за аналогією з https://www.goodyear.eu/uk_ua/consumer.html, адаптовані під бренд Bridgestone, український ринок та одну мовну локаль.

**Цілі сайту:**

- Іміджеве представлення бренду Bridgestone в Україні.
- Надання зручної та зрозумілої інформації про шини для кінцевих клієнтів.
- Допомога користувачу у виборі шин (за розміром, за маркою та моделлю автомобіля).
- Направлення користувача до дилерів / партнерів (карта дилерів, контакти).

**Цільова аудиторія:**

- Водії легкових і легких комерційних автомобілів в Україні.
- Власники автопарків малого та середнього бізнесу (як вторинна ЦА).
- Вік 25–60, основна мова споживання контенту — українська 

---

## 2. Мови та регіони

- **Мова інтерфейсу:** тільки українська.
- **Локалізація:** весь текстовий, графічний і допоміжний контент — українською мовою.
- Передбачити можливість у майбутньому додати додаткові мови (архітектурно передбачити мультимовність: зберігання текстів з прив'язкою до коду мови, але на старті — один інтерфейс).

## 3. Структура сайту (інформаційна архітектура)

### 3.1. Основні розділи навігації (верхнє меню)

Попередня структура меню:

1. Головна
2. Шини для легкових авто
3. Шини для позашляховиків та кросоверів (SUV/4x4)
4. Шини для легких комерційних авто (LCV) — за потреби
5. Пошук шин
   - За розміром
   - За маркою та моделлю авто
6. Дилери / Де купити
7. Про бренд Bridgestone
8. Технології та інновації
9. Корисна інформація / Поради (опційно — блог/статті)
10. Контакти

Точний склад і назви пунктів меню уточнюються на етапі прототипування, але загальний обсяг розділів має бути співставним із референсом Goodyear, адаптованим під лінійку Bridgestone.

### 3.2. Головна сторінка

Основні блоки:

- Великий іміджевий банер з візуалом Bridgestone (сезонні акценти: зима / літо / всесезонні).
- Швидкий пошук шин (міні-форма):
  - Вкладка «За розміром»
  - Вкладка «За авто»
- Промо-блоки:
  - Посилання на основні категорії шин (літні, зимові, всесезонні).
  - Посилання на популярні моделі шин.
- Блок «Чому Bridgestone» — коротко про переваги бренду.
- Блок «Технології та інновації».
- Блок «Знайти дилера поруч» (карта / пошук за містом).
- Блок новин або статей (за наявності контент‑плану).

## 4. Функціональні вимоги

### 4.1. Каталог шин

**Структура каталогу:**

- Поділ за:
  - Типом транспорту: легкові, SUV/4x4, легкі комерційні (опційно).
  - Сезонністю: літні, зимові, всесезонні.
- Сторінка категорії з фільтрами та списком моделей шин.

**Картка моделі шини:**

- Назва моделі.
- Тип (літня / зимова / всесезонна).
- Тип застосування (легкове авто / SUV / LCV).
- Короткий опис (ключові переваги).
- Технічні характеристики:
  - Доступні розміри (список / таблиця з фільтром).
  - Індекси навантаження та швидкості.
  - Маркування EU (рівень шуму, зчеплення на мокрій дорозі, паливна ефективність — за потреби).
- Фотографії / рендери шини.
- Блок «Для яких умов підходить» (місто, траса, зима, офф‑роуд тощо).
- Кнопка/посилання «Знайти дилера» (веде до розділу дилерів, за можливості — з передачею моделі як параметра).
- Опційно: блок «Рекомендовані шини» (схожі моделі).

**Фільтри в каталозі:**

- Сезон (літо / зима / всесезон).
- Тип авто (легкове / SUV / LCV).
- Ширина профілю, висота профілю, діаметр (якщо користувач зайшов не через конкретний пошук за розміром).
- Діапазон ціни (якщо ціни відображаються; на старті можливий варіант без публічних цін).
- Новинки / популярні моделі (сортування).

### 4.2. Пошук шин за розміром

**Форма:**

- Поля:
  - Ширина (dropdown).
  - Висота профілю (dropdown).
  - Діаметр (dropdown).
  - Сезон (опційно).
- Кнопка «Знайти шини».
- Валідація: обов'язкові поля, коректність формату.

**Результат пошуку:**

- Сторінка зі списком відповідних шин (моделей / конкретних типорозмірів).
- Можливість перейти до картки моделі.
- Можливість одразу перейти до «Знайти дилера» (брендовий сценарій: користувач обрав шину — далі йому потрібен продавець).

### 4.3. Пошук шин за маркою та моделлю автомобіля

**Форма:**

- Кроки / поля:
  - Марка авто (dropdown).
  - Модель авто (динамічний dropdown залежно від марки).
  - Рік випуску.
  - Тип кузова (якщо є в структурі даних).
- Опційно окремий крок «Тип шини» (літня / зимова / всесезонна).

**Результат:**

- Список рекомендованих типорозмірів і відповідних моделей шин Bridgestone.
- Пояснення формату: «Рекомендований розмір для вашого авто».
- Перехід до картки моделі або одразу до карти дилерів.

### 4.4. Карта дилерів / Де купити

**Функціонал:**

- Інтерактивна карта (Google Maps / OpenStreetMap), відображення точок дилерів / партнерів.
- Фільтри:
  - За містом / областю.
  - За типом точки (офіційний дилер / партнер / сервісний центр тощо — якщо є класифікація).
- Пошук за адресою / містом (input + автодоповнення за містами).
- При кліку на точку:
  - Назва дилера.
  - Адреса.
  - Телефон.
  - Посилання на сайт (якщо є).
  - Години роботи (опційно).
- Опційно: кнопка «Маршрут» (відкриває зовнішній сервіс побудови маршруту).

**Адмін‑частина для дилерів:**

- Можливість додавати, редагувати та видаляти записи дилерів (через CMS / адмінку).
- Поля дилера: назва, адреса, координати (широта / довгота), контакти, тип.

## 7. Адміністративна панель / CMS

### 7.1. Загальні вимоги

- Зручна CMS для:
  - Управління каталогом шин (моделі, розміри, характеристики).
  - Управління дилерами (адреси, координати тощо).
  - Редагування контентних сторінок: Про бренд, Технології, Корисна інформація, Головна.
- Розмежування ролей:
  - Адміністратор (повні права).
  - Контент‑редактор (редагування контенту без доступу до технічних налаштувань).

### 7.2. Операції в адмін‑частині

- Створення / редагування / видалення моделі шини.
- Прив'язка типорозмірів до моделей (таблично).
- Завантаження зображень (з автоматичним ресайзом / оптимізацією).
- Управління списком марок / моделей авто та їх рекомендованими типорозмірами (якщо використовується власна база).
- Управління дилерами.
- Редагування текстових блоків і SEO‑метаданих сторінок.

## 8. Дизайн і UX

### 8.1. Загальні принципи

- Відповідність бренд‑гайду Bridgestone:
  - Кольорова палітра, шрифти, логотипи.
  - Використання фірмових елементів, без копіювання стилю Goodyear (лише логіка та UX).
- Сучасний, «легкий» інтерфейс із акцентом на зручність вибору шин.

### 8.2. Адаптивність

- Повна адаптивність для:
  - Desktop (≥ 1200 px).
  - Tablet (768–1199 px).
  - Mobile (до 767 px).
- Мобільна версія не повинна бути спрощеною на шкоду функціоналу: пошук за розміром / за авто, каталог і карта дилерів мають бути повністю доступними.

### 8.3. UX‑вимоги

- Максимум 2–3 кроки від головної сторінки до результату пошуку шин.
- Зрозуміла структура фільтрів, відсутність перевантаження.
- «Хлібні крихти» для внутрішніх сторінок (каталог, статті).
- Єдиний стиль кнопок і інтерактивних елементів.

## 9. SEO та технічні вимоги

### 9.1. SEO‑основа

- Людинозрозумілі URL (ЧПУ):
  - Для моделей шин: `/shyny/<category>/<model-name>` (український трансліт, формат уточнюється).
  - Для статей: `/blog/<article-slug>`.
- Налаштовувані метатеги title та description для:
  - Головної.
  - Категорій шин.
  - Моделей шин.
  - Контентних сторінок.
  - Статей.
- Структуровані дані (schema.org) для:
  - Продуктів (моделі шин).
  - Локальних компаній (дилери) — за потреби.

### 9.2. Продуктивність

- Оптимізація зображень (webp/avif за можливості).
- Мініфікація CSS та JS.
- Lazy‑loading для зображень (особливо в каталозі).
- Орієнтовні показники:
  - LCP < 2–2.5 секунди на 4G.
  - Мінімально можливий розмір завантажуваних ресурсів для першого екрану.

### 9.3. Безпека

- HTTPS за замовчуванням.
- Захист адмін‑панелі:
  - Складні паролі.
  - Опційно — обмеження доступу за IP.
- Актуальні версії бібліотек / фреймворків.

## 10. Технічний стек (рекомендації)

Остаточний вибір стеку погоджується окремо, але базові рекомендації:

- **Frontend:**
  - Сучасний фреймворк (React/Next.js, Vue/Nuxt) або класичний стек (HTML+CSS+JS) при простому бекенді.
  - Адаптивна верстка (flex/grid), без застарілих залежностей.
- **Backend / CMS:**
  - Готова CMS (WordPress із кастомним шаблоном, headless CMS, Strapi тощо)
  - або кастомне рішення на сучасному фреймворку (Laravel, Symfony, Django).
- **База даних:**
  - Реляційна БД (MySQL/PostgreSQL) для зберігання каталогу шин, дилерів та контенту.

1. **Аналітика та уточнення вимог**
   - Деталізація поточного ТЗ.
   - Збір бренд‑матеріалів, гайдів, прикладів.
2. **Проєктування**

3. **Дизайн**
   - Концепт.
   - Дизайн усіх ключових екранів + адаптиви.
4. **Розробка**
   - Frontend.
   - Backend / CMS.
   - Інтеграція аналітики.
5. **Тестування**
   - Функціональне (пошук, фільтри, карта).

6. **Наповнення**
