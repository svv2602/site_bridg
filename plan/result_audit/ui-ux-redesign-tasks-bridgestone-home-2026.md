# План внедрения UI/UX‑редизайна Bridgestone Україна

Этот документ переводит результаты аудита в конкретные этапы и задачи для фронтенда.

## 1. Общие цели редизайна

- Укрепить ощущение международного бренда Bridgestone (цвет, типографика, визуал).
- Упростить путь к подбору шин и поиску дилера.
- Сделать главный экран более фокусным и эмоциональным.
- Синхронизировать фронтенд с дизайн‑системой и чек-листами в репозитории.

## 2. Фаза 1 — Дизайн‑система и токены

**Цель:** зафиксировать обновлённую палитру, типографику и базовые отступы, не ломая существующую структуру.

### 2.1. Цвета и токены

- [ ] Перепроверить и при необходимости скорректировать цветовые токены в [`globals.css`](frontend/src/app/globals.css:1) под аудит:
  - [ ] Проверить значения primary/secondary и stone‑палитры.
  - [ ] Ограничить использование Bridgestone Red только для CTA, бейджей и ключевых маркеров.
- [ ] Вынести в отдельный блок в [`DESIGN_SYSTEM.md`](frontend/DESIGN_SYSTEM.md:1) актуальные значения цветов из [`globals.css`](frontend/src/app/globals.css:1) (чтобы документ и код были синхронизированы).

### 2.2. Типографика

- [ ] Определить целевую шрифтовую пару (Inter / Manrope / IBM Plex Sans или оставить Geist) и зафиксировать её в [`DESIGN_SYSTEM.md`](frontend/DESIGN_SYSTEM.md:58).
- [ ] Описать итоговую иерархию заголовков и текста (Hero H1, H2, H3, body, caption) с классами Tailwind и примерами.
- [ ] Проверить текущую реализацию масштабов в [`globals.css`](frontend/src/app/globals.css:85) и привести в соответствие с описанной иерархией.

### 2.3. Сетка и отступы

- [ ] Обновить или описать паттерн секционных отступов (section-spacing) в [`globals.css`](frontend/src/app/globals.css:464).
- [ ] Зафиксировать в [`DESIGN_SYSTEM.md`](frontend/DESIGN_SYSTEM.md:92) фактические значения max-width контейнера и горизонтальных отступов.
- [ ] Определить отдельные пресеты для:
  - [ ] Hero‑секций (более крупные отступы).
  - [ ] Контентных блоков (средние).
  - [ ] Вспомогательных секций (меньшие).

## 3. Фаза 2 — Навигация и header

**Цель:** сделать структуру сайта прозрачной и снизить перегруз верхней части страницы.

### 3.1. Основной header

- [ ] Проработать новый layout для [`MainHeader`](frontend/src/components/MainHeader.tsx:18):
  - [ ] На десктопе вывести основные пункты меню в линию рядом с логотипом (без обязательного открытия бургера).
  - [ ] Оставить бургер как доступ к полному меню на мобильных и планшетах.
- [ ] Разделить визуально бренд‑зону (логотип + подпись) и навигацию.
- [ ] Проверить контраст и размеры текста в шапке для лучшей читабельности.

### 3.2. Верхний топ‑бар

- [ ] Пересмотреть состав элементов в верхнем баре в [`RootLayout`](frontend/src/app/layout.tsx:115):
  - [ ] Сохранить только наиболее важные для первого экрана элементы (язык, toggle темы).
  - [ ] Перевести второстепенные (телефон, «знайти дилера поруч») в более спокойный визуальный стиль.
- [ ] Проверить адаптивное поведение топ‑бара: не должен занимать слишком много вертикального места на мобильных.

## 4. Фаза 3 — Hero‑секция главной страницы

**Цель:** сделать первый экран фокусным, эмоциональным и сразу ведущим к подбору шин.

### 4.1. Контент и структура hero

- [ ] На уровне контента сформулировать финальный заголовок и подзаголовок hero (уточнить с бизнесом).
- [ ] Добавить бейдж доверия (офіційний сайт / безпека на кожен сезон).
- [ ] В hero вынести две главные CTA:
  - [ ] «Підібрати шини» (якорь на форму или встроенная форма).
  - [ ] «Знайти дилера поруч».

### 4.2. Визуал и фон

- [ ] Спроектировать структуру hero в [`Home`](frontend/src/app/page.tsx:72) с двумя колонками:
  - [ ] Слева — текстовый блок и CTA.
  - [ ] Справа — визуал + форма швидкого пошуку.
- [ ] Использовать существующие утилиты hero (`hero-dark`, `hero-glow`, `glass`) в [`globals.css`](frontend/src/app/globals.css:341) и при необходимости доработать их.
- [ ] Определить временные placeholder‑изображения и подготовить места для будущих реальных фото (через `next/image`).

### 4.3. Форма быстрого подбора шин

- [ ] Проверить и обновить UX формы [`QuickSearchForm`](frontend/src/components/QuickSearchForm.tsx:1) (если компонент уже реализован):
  - [ ] Минимизировать количество полей на первом шаге.
  - [ ] Добавить понятные подписи и плейсхолдеры.
- [ ] Визуально оформить форму как стеклянную карточку на тёмном фоне hero.

## 5. Фаза 4 — Контентные блоки главной: преимущества, категории, популярные модели

**Цель:** упростить восприятие иерархии блоков и сделать карточки более «брендовыми».

### 5.1. Блок «Чому Bridgestone» (преимущества)

- [ ] Перепроверить контент и количество преимуществ в блоке `features` в [`Home`](frontend/src/app/page.tsx:37).
- [ ] Упростить визуал карточек: меньше тяжёлых теней, больше воздуха, один главный акцент (иконка + заголовок).
- [ ] Проверить размер иконок и текстов для равновесия (не перегружать графикой).

### 5.2. Блок «Шини за сезоном/типом авто»

- [ ] Пересобрать карточки `tyreCategories` в [`Home`](frontend/src/app/page.tsx:10) с акцентом на:
  - [ ] Чёткий заголовок категории.
  - [ ] Краткое объяснение, для кого и для яких умов.
  - [ ] CTA «Перейти» в одном стиле для всех карточек.
- [ ] Протестировать, чтобы блок хорошо читался и на мобильных (по одному элементу в ряд).

### 5.3. Блок «Популярні моделі»

- [ ] Проверить алгоритм отбора `featuredTyres` в [`Home`](frontend/src/app/page.tsx:75) (isPopular, лимит 3 шт.).
- [ ] Обновить визуальный паттерн карточек популярных моделей:
  - [ ] Название модели.
  - [ ] Бейдж сезон/тип авто.
  - [ ] 1–2 ключевых преимущества.
  - [ ] CTA «Детальніше» + «Знайти дилера».

## 6. Фаза 5 — Блок статей и доверия

**Цель:** показать, что бренд не только продаёт шины, но и допомагає водіям розбиратися в темі.

### 6.1. Блок статей на главной

- [ ] Выровнять карточки статей в [`Home`](frontend/src/app/page.tsx:210) по единому шаблону:
  - [ ] Бейдж категории.
  - [ ] Заголовок.
  - [ ] Краткий опис.
  - [ ] Время чтения.
- [ ] Проверить читаемость заголовков и описаний на мобильных.

### 6.2. Блок доверия

- [ ] Добавить отдельный блок доверия (логотипи ADAC, AutoBild, TCS, TyreReviews) на главную страницу:
  - [ ] Кратко объяснить, що Bridgestone опирається на незалежні тести та огляди.
  - [ ] Возможно, использовать уже существующие логотипы в [`public/images/logos`](frontend/public/images/logos/adac.svg:1).

## 7. Фаза 6 — Финальный полиш, доступность и тестирование

**Цель:** довести интерфейс до уровня production‑качества.

### 7.1. Адаптив и поведение

- [ ] Пройтись по основным брейкпоинтам (мобильный, планшет, десктоп) для главной и связанных страниц.
- [ ] Убедиться, что hero и ключевые CTA всегда остаются видимыми и удобными для клика/тача.

### 7.2. Доступность

- [ ] Проверить цветовой контраст текста и фона (особенно на тёмных секциях).
- [ ] Убедиться, что фокус‑состояния (реализованные в [`globals.css`](frontend/src/app/globals.css:575)) корректно видны на всех ключевых элементах.
- [ ] Проверить управление клавиатурой для навигации и форм.

### 7.3. Проверка сценариев

- [ ] Пройти ключевые пользовательские сценарии:
  - [ ] Підбір шин с главной.
  - [ ] Пошук дилера.
  - [ ] Переход к статте и обратно.
- [ ] Зафиксировать найденные UX‑проблемы и завести отдельные задачи в таск‑трекер.
