# Інструкція для ІІ-агента

## 1. Структура папки

Створи папку `{task-name}/` з наступними файлами:

```
{task-name}/
├── README.md              # Огляд задачі, цілі, критерії успіху
├── PROGRESS.md            # Поточний статус (оновлюється після кожної задачі)
├── phase-01-{назва}.md    # Перша фаза
├── phase-02-{назва}.md    # Друга фаза
└── phase-NN-{назва}.md    # N-на фаза
```

## Правила перевикористання коду

### ОБОВ'ЯЗКОВО перед реалізацією:
1. **Пошук існуючого функціоналу** - перед написанням нового коду ЗАВЖДИ шукай схожий існуючий код
2. **Аналіз патернів** - вивчи як реалізовані схожі фічі в проекті
3. **Перевикористання компонентів** - використовуй існуючі компоненти, хуки, утиліти

### Де шукати:
```
frontend/src/
├── components/          # UI компоненти (MainHeader, ThemeToggle, AnimatedMain)
├── app/                 # Сторінки - референс для нових сторінок
│   ├── page.tsx         # Головна - патерн hero + секції
│   ├── tyre-search/     # Форми пошуку - патерн табів
│   ├── dealers/         # Списки з фільтрами
│   └── shyny/[slug]/    # Динамічні сторінки
└── lib/
    ├── data.ts          # Типи даних та mock-дані
    └── api/             # API-шар для CMS інтеграції
```

## Правила інтеграції з CMS/API

### Поточний стан:
- Дані зберігаються в `lib/data.ts` як mock
- API-шар підготовлений в `lib/api/`
- Функції повертають mock-дані, але готові до заміни

### При додаванні нових даних:
1. Додай тип в `lib/data.ts`
2. Додай mock-дані в `MOCK_*` константи
3. Створи API-функції в `lib/api/`
4. Використовуй API-функції в компонентах (не mock напряму)

### Чекліст:

## 5. Протокол роботи агента

### При початку сесії
1. Прочитай `PROGRESS.md` щоб визначити поточний стан
2. Відкрий файл поточної фази
3. Знайди першу незавершену задачу

### При початку нової фази (КРИТИЧНО ВАЖЛИВО)
1. **ПЕРША задача фази - ЗАВЖДИ "N.0 Аналіз та планування"**
2. **A. Аналіз коду:** Пошук існуючого функціоналу, вивчення патернів
3. **B. Аналіз залежностей:** Визнач потрібні типи, API, компоненти
4. **C. Перевірка дизайну:** Обери референс-сторінку для стилів
5. Заповни всі нотатки в задачі N.0
6. Тільки після цього переходь до реалізації

### При виконанні задачі
1. **Перед написанням коду** - перевір нотатки з задачі N.0
2. Виконай задачу, перевикористовуючи знайдені патерни
3. Одразу відміть чекбокс як [x]
4. Додай нотатки якщо потрібно
5. При необхідності вкажи змінені файли

### При завершенні фази
1. Перевір всі задачі [x]
2. Онови статус фази
3. Зроби коміт з повідомленням формату:
   `checklist({task-name}): phase-N {назва} completed`
4. Онови PROGRESS.md
5. Переходь до наступної фази

### При збої/перериванні
1. Збережи поточний прогрес (відміть виконані задачі)
2. Онови PROGRESS.md з поточним статусом
3. При відновленні - почни з читання PROGRESS.md
