# Унификация цветовой системы Bridgestone

## Цель
Привести цветовую систему проекта к единой консистентной архитектуре:
- Полный отказ от `zinc-*` в пользу семантических токенов
- Устранение прямых hex-значений в компонентах
- Синхронизация документации с реальной системой токенов
- Консистентный опыт в Light и Dark темах

## Критерии успеха
- [x] `DESIGN_SYSTEM.md` соответствует фактическим токенам в `globals.css`
- [x] Нет использования `zinc-*` классов в UI-слое (кроме dev-утилит)
- [x] Нет прямых hex-значений для нейтральных цветов в компонентах
- [x] Админка использует те же токены, что и публичная часть
- [x] Hover/focus/disabled состояния работают через токены
- [x] Контраст текста соответствует WCAG AA в обеих темах

## Фазы работы
1. **Документация** - Обновление DESIGN_SYSTEM.md под фактическую систему
2. **Токен-система** - Добавление недостающих токенов в globals.css
3. **Миграция админки** - Замена zinc на семантические токены
4. **Публичные компоненты** - Миграция MainHeader и других компонентов
5. **Валидация** - Проверка обеих тем, контраста, состояний

## Правила цветовой системы

### Иерархия фонов (Light):
| Уровень | Токен | Назначение |
|---------|-------|------------|
| Page | `bg-background` / `--stone-50` | Фон страницы |
| Section Alt | `bg-stone-100` | Чередование секций |
| Card | `bg-card` + `border-border` | Карточки |
| Interactive | hover/focus состояния | Интерактивные элементы |

### Иерархия фонов (Dark):
| Уровень | Токен | Назначение |
|---------|-------|------------|
| Page | `bg-background` / `--black-base` | Фон страницы |
| Section Alt | `--wet-asphalt` | Чередование секций |
| Card | `--graphite` + `border-dark` | Карточки |
| Interactive | `--graphite-hover` | Интерактивные элементы |

### Запрещенные паттерны:
- `bg-zinc-*`, `text-zinc-*`, `border-zinc-*` в UI-компонентах
- Прямые hex (#0c0a09, #1c1917) для нейтральных цветов
- `btn-brand` для стандартных CTA (только для логотипа/алертов)
