# Goodyear UX Improvements

## Ціль
Покращити UX сайту Bridgestone UA на основі аналізу референсного сайту Goodyear EU (https://www.goodyear.eu/uk_ua/consumer.html). Впровадити кращі практики UX, що зроблять сайт більш привабливим та зручним для користувачів.

## Джерело вимог
Порівняльний аналіз сайтів Goodyear EU vs Bridgestone UA, проведений 2026-01-13.

## Критерії успіху
- [ ] Hero-секція містить реальні фото шин замість абстрактних іконок
- [ ] Карусель популярних продуктів на головній сторінці
- [ ] Мега-меню з категоріями шин при hover
- [ ] Візуальні блоки по типах авто (легкові, SUV, комерційні)
- [ ] Локатор дилерів інтегрований на головну сторінку
- [ ] Емоційні заголовки замість інформативних
- [ ] Footer з соціальними мережами та Instagram

## Фази роботи
1. **Hero з фото шин** - заміна абстрактної іконки на реальне фото флагманської шини
2. **Карусель продуктів** - Swiper/Embla карусель популярних моделей після hero
3. **Мега-меню** - розгалужене меню з категоріями при hover
4. **Блоки по типах авто** - візуальні картки для легкових/SUV/комерційних
5. **Локатор дилерів** - компактний блок пошуку дилерів на головній
6. **UX покращення** - емоційні заголовки, сезонність, переключатель Car/Truck
7. **Footer та соцмережі** - Instagram feed, посилання на соцмережі

## Правила перевикористання коду

### ОБОВ'ЯЗКОВО перед реалізацією:
1. **Пошук існуючого функціоналу** - перед написанням нового коду ЗАВЖДИ шукай схожий існуючий код
2. **Аналіз патернів** - вивчи як реалізовані схожі фічі в проекті
3. **Перевикористання компонентів** - використовуй існуючі компоненти, хуки, утиліти

### Де шукати:
```
frontend/src/
├── components/          # UI компоненти
│   ├── MainHeader.tsx   # Поточне меню - модифікувати для мега-меню
│   ├── SeasonalHero.tsx # Поточний hero - модифікувати для фото
│   ├── TyreCard.tsx     # Картки шин - використати в каруселі
│   ├── DealersMap.tsx   # Карта дилерів - адаптувати для головної
│   └── QuickSearchForm.tsx # Форма пошуку - референс для стилів
├── app/
│   ├── page.tsx         # Головна - основний файл для змін
│   ├── dealers/         # Сторінка дилерів - референс
│   └── passenger-tyres/ # Каталог - референс для списків
└── lib/
    ├── api/tyres.ts     # API для шин
    └── api/dealers.ts   # API для дилерів
```

### Існуючі патерни стилів:
| Елемент | Класи Tailwind |
|---------|----------------|
| Hero секція | `hero-dark relative border-b border-hero-border py-8 md:py-12 hero-glow hero-grid-pattern` |
| Картка | `rounded-2xl border border-border bg-card p-6 shadow-lg` |
| Primary кнопка | `rounded-full bg-primary px-6 py-3 text-sm font-semibold text-white` |
| Secondary кнопка | `rounded-full border border-border bg-transparent px-6 py-3` |
| Анімована картка | `<AnimatedCard delay={idx * 0.1}>` |

## Правила дизайну

### Колірна палітра (НЕ змінювати):
- Primary: `#E30613` (Bridgestone Red)
- Background: `stone-50` (#fafaf9)
- Cards: `#ffffff` з border `stone-200`
- Dark hero: градієнт `stone-950 → stone-800`

### Шрифти:
- Geist Sans
- Мінімальний розмір: 14px

### Border-radius:
- Картки: `rounded-2xl` (20px)
- Кнопки: `rounded-full`
- Inputs: `rounded-xl` (12px)

## Технічні вимоги

### Карусель:
- Використати Embla Carousel (вже в проекті) або Swiper
- Responsive: 1 слайд на mobile, 2-3 на tablet, 4 на desktop
- Автопрокрутка з паузою на hover
- Swipe-підтримка для touch devices

### Мега-меню:
- Показувати при hover з затримкою 150ms
- Анімація fade-in
- Закривати при mouseleave
- Підтримка клавіатурної навігації

### Локатор дилерів:
- Компактна версія без повної карти
- Тільки поле пошуку + 3-4 найближчих результати
- Кнопка "Показати на карті" → /dealers
