# Дизайн‑система Bridgestone Україна (фронтенд‑прототип)

Ця дизайн‑система описує принципи оформлення фронтенду сайту Bridgestone Україна.
Вона орієнтована на:

- бренд Bridgestone (червоний / чорний / білий, строгість, технічність);
- сучасний, «легкий» вигляд із акцентом на великий візуал (як у Continental/Goodyear);
- просту, читабельну інформаційну архітектуру з чіткими блоками.

Документ використовується як орієнтир для компонування та стилізації сторінок у
Next.js‑проєкті [`frontend`](./).

---

## 1. Палітра кольорів

Базова ідея: сильний контраст чорний/білий + акцентний червоний Bridgestone,
нейтральні сіри для фону/бордерів.

> Точні бренд‑коди Bridgestone можна буде замінити після отримання брендбуку.

### Primary

- **Primary Red** — акцентний колір (кнопки, бейджі, ключові CTA)
  - `#E30613` (умовний Bridgestone Red)
- **Primary Black** — основний колір для темних елементів
  - `#111111`

### Background / Surface

- **Background Light** — фон сторінок
  - `#F4F4F5` (близько до Tailwind zinc‑100)
- **Surface White** — картки, контентні блоки
  - `#FFFFFF`
- **Surface Dark Hero** — темні hero‑секції
  - `#0B0B0C` – градієнти через близькі значення: `from-zinc-900 to-zinc-800`

### Text

- **Text Primary** — базовий текст
  - `#111827` (темний, високий контраст)
- **Text Secondary** — другорядний опис
  - `#4B5563`
- **Text Muted** — допоміжні підписи, примітки
  - `#6B7280`
- **Text On Dark** — текст на темному фоні
  - `#F9FAFB`

### Borders / Lines

- **Border Light** — бордери карток, роздільники
  - `#E5E7EB`
- **Border Muted / Dashed** — інформаційні бокси
  - `#D4D4D8`

## 2. Типографіка

Фронтенд зараз використовує шрифт Geist (системно підключений у Next). У
продакшн‑версії можна замінити на офіційно дозволений бренд‑шрифт Bridgestone
або web‑альтернативу (наприклад, Inter/Roboto).

### Ієрархія заголовків (приблизно Tailwind)

- **Hero Title (H1)**:
  - клас: `text-3xl md:text-4xl font-semibold tracking-tight`
  - використання: головний слоган на головній, верхні заголовки розділів.
- **Section Title (H2)**:
  - клас: `text-lg md:text-xl font-semibold`
  - використання: заголовки блоків «Чому Bridgestone», «Технології», «Популярні моделі».
- **Card Title (H3)**:
  - клас: `text-sm font-semibold`
  - використання: назви моделей шин, статей, дилерів.

### Текст

- **Body текст**:
  - клас: `text-sm text-zinc-600 leading-relaxed`
  - довгі абзаци на світлому фоні.

- **Малі підписи / labels**:
  - клас: `text-xs text-zinc-500 uppercase tracking-wide`
  - для бейджів «Стаття», «Офіційний дилер», типів шин, сезонності.

- **Caption / примітки**:
  - клас: `text-[11px] text-zinc-500`
  - допоміжний текст у формі, disclaimers, примітки до прототипу.

---

## 3. Сітка та відступи

- Максимальна ширина контенту: `max-w-6xl` (≈ 1152px).
- Горизонтальні відступи:
  - мобільний: `px-4`
  - десктоп: `md:px-6`
- Вертикальні блоки:
  - великі секції: `py-10 md:py-12`
  - всередині карток: `p-4 md:p-5`

### Секційний патерн

1. **Hero‑секції**:
   - фон: темний градієнт (`bg-gradient-to-br from-zinc-900 via-zinc-900 to-zinc-800`)
   - текст: білий/світлий.
   - компонування:
     - зліва: заголовок + bullets + CTA;
     - справа: карта / форма пошуку / великий візуал (placeholder під зображення шини/авто).

2. **Контентні блоки**:
   - фон: білий (`bg-white`) з бордерами `border-zinc-200`.
   - тінь: помірна (`shadow-sm`), щоб не перевантажувати.

3. **Альтернативні секції**:
   - фон: світло‑сірий (`bg-zinc-50`) для розділення інформаційних блоків.

---

## 4. Візуальні елементи та зображення

Цілі:

- зробити шини та авто «героями» сторінок;
- уникати перевантажених фонових елементів;
- використовувати великі фотографії на першому екрані та в ключових блоках.

### Hero зображення

- Для головної:
  - 1–2 основні hero‑зображення (шини на авто в реальному середовищі — місто/траса/зима).
  - В коді зараз стоять placeholder‑блоки (`div` з темним фоном), куди можна
    підставити `next/image` з реальними фото.
- Для каталогу:
  - невеликі круглі «плашки» з силуетом шини замість реального фото (зараз)
  - в продакшн — заміна на офіційні рендери.

### Патерни зображень

- **Картки шин**:
  - круглий/квадратний thumbnail (40–64px) з фото шини.
- **Технології**:
  - іконка технології (SVG) + короткий опис.
- **Брендові блоки**:
  - широкі банери (full‑width в межах `max-w-6xl`) з overlay‑текстом.

## 5. Компоненти UI

### Кнопки

- **Primary CTA** (темна):
  - `inline-flex items-center rounded-full bg-zinc-900 px-4 py-2 text-xs font-semibold text-white hover:bg-zinc-800`
  - використовувати для: «Знайти шини», «Підібрати шини», «Показати дилерів».

- **Secondary CTA** (outline):
  - `inline-flex items-center rounded-full border border-zinc-200 px-3 py-1 text-xs hover:bg-zinc-50`
  - для: «Детальніше», «Читати статтю», другорядні дії.

- **Accent CTA** (червоний, за потреби):
  - `bg-red-500 hover:bg-red-600 text-white rounded-full text-xs px-4 py-2`
  - помірно використовувати для ключових маркетингових дій.

### Картки

- **Картка шини**:
  - контейнер: `rounded-xl border border-zinc-200 bg-white p-4 shadow-sm`
  - layout:
    - маленький круглий блок для фото/іконки;
    - назва моделі + бейдж сезону/типу авто;
    - короткий опис;
    - список типорозмірів як pills;
    - кнопки дій.

- **Картка дилера**:
  - акцент на назві, типі точки, адресі.
  - чіткий текстовий блок, без зайвих декоративних елементів.

- **Картка статті**:
  - невеликий card‑layout, чіткий заголовок, короткий прев’ю‑текст, теги.

---

## 6. Адаптивність

- Мобайл:
  - одноколонна сітка;
  - пріоритет форм (пошук шин, пошук дилера);
  - горизонтальні відступи `px-4`.
- Планшет:
  - 2‑колонні секції (форма + текст, список + карта).
- Десктоп:
  - 2–3 колонки у каталогах/статтях;
  - hero з двома колонками (текст + зображення/форма).

---

## 7. Взаємодія з референсами

- **Не копіюємо** Continental чи Goodyear, але:
  - беремо від них:
    - чистоту першого екрана;
    - великі зображення;
    - прості зрозумілі блоки з 2–3 ключовими меседжами;
  - накладаємо на брендові кольори та строгішу типографіку Bridgestone.

- **Від firststop.be / bridgestone.com**:
  - беремо лише бренд‑ідентику (логотип, дозволені кольори/патерни — після отримання гайду).
  - не наслідуємо legacy‑мікси layout’ів.

1. Скорегувати кольори (primary/secondary, градієнти).
2. За потреби замінити шрифт Geist на офіційний веб‑шрифт Bridgestone.
3. Підставити реальні зображення замість placeholder‑блоків у hero/картках.
