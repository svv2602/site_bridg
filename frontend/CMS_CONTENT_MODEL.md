# Модель контенту для headless CMS (Bridgestone Україна)

Цей документ описує рекомендовану структуру контент-типів для headless CMS
типу Strapi / Directus / іншої системи, яка буде джерелом даних для
фронтенду на Next.js.

Базові принципи:

- Окремий тип для **моделей шин** (TyreModel).
- Окремий тип для **типорозмірів** (TyreSize) з прив’язкою до моделі.
- Окремий тип для **дилерів** (Dealer).
- Окремий тип для **статей/порад** (Article).
- Окремий тип для **технологій** (Technology).
- Окремий тип або словник для **фітментів авто** (VehicleFitment) —
  прив’язка марка/модель/рік → рекомендовані типорозміри.

## 1. TyreModel (модель шини)

Колекційний тип, один запис = одна модель шини (наприклад,
“Bridgestone Turanza T005”).

| Поле          | Ключ            | Тип            | Обов’язково | Коментар                                   |
|--------------|-----------------|----------------|-------------|--------------------------------------------|
| Назва        | name            | String         | так         | Маркетингова назва моделі                  |
| Слаг URL     | slug            | UID/Slug       | так         | Унікальний slug для URL `/shyny/...`       |
| Сезон        | season          | Enum           | так         | `summer` / `winter` / `allseason`          |
| Тип авто     | vehicleTypes    | Enum (multi)   | так         | `passenger`, `suv`, `lcv`                  |
| Новинка      | isNew           | Boolean        | ні          | Позначка “новинка”                         |
| Популярна    | isPopular       | Boolean        | ні          | Для сортування / виділення у каталозі      |
| Короткий опис| shortDescription| Text           | так         | 1–3 речення про переваги моделі            |
| Опис повний  | description     | Rich text      | ні          | Детальний опис (опційно)                   |
| EU-маркування| euLabel         | Component      | ні          | Вологе зчеплення, економія, шум            |
| Розміри      | sizes           | Relation (1–N) | так         | Зв’язок з TyreSize                          |
| Умови        | usage           | JSON/Component | ні          | Місто/траса/зима/офф-роуд                  |
| Технології   | technologies    | Relation (M–N) | ні          | Зв’язок з Technology                        |
| Зображення   | images          | Media (multi)  | ні          | Фото/рендери шини                          |
| SEO title    | seoTitle        | String         | ні          | Персональний title для моделі              |
| SEO desc     | seoDescription  | Text           | ні          | meta description                            |

Компонент **euLabel** (вкладений тип):

| Поле             | Ключ           | Тип     | Коментар          |
|-----------------|----------------|---------|-------------------|
| Вологе зчеплення| wetGrip        | Enum    | A–E               |
| Економія пального| fuelEfficiency| Enum    | A–E               |
| Рівень шуму, dB | noiseDb        | Integer | Наприклад, 71     |

## 2. TyreSize (типорозмір шини)

Колекційний тип, один запис = один типорозмір, прив’язаний до моделі.

| Поле           | Ключ        | Тип      | Обов’язково | Коментар          |
|---------------|------------|----------|-------------|-------------------|
| Модель шини   | tyreModel  | Relation | так         | Посилання на TyreModel |
| Ширина        | width      | Integer  | так         | Наприклад, 205    |
| Висота профілю| aspectRatio| Integer  | так         | Наприклад, 55     |
| Діаметр       | diameter   | Integer  | так         | Наприклад, 16     |
| Індекс навант.| loadIndex  | Integer  | ні          | Наприклад, 91     |
| Індекс швидк. | speedIndex | String   | ні          | Наприклад, V      |

## 3. Dealer (дилер / партнер)

Колекційний тип для карти та списку дилерів.

| Поле          | Ключ        | Тип      | Обов’язково | Коментар                         |
|--------------|------------|----------|-------------|----------------------------------|
| Назва        | name       | String   | так         | Назва дилера/партнера           |
| Тип точки    | type       | Enum     | так         | `official` / `partner` / `service` |
| Місто        | city       | String   | так         | Місто                           |
| Адреса       | address    | String   | так         | Вулиця, будинок                 |
| Широта       | latitude   | Float    | ні          | Для карти                        |
| Довгота      | longitude  | Float    | ні          | Для карти                        |
| Телефон      | phone      | String   | ні          | +380 ...                         |
| Сайт         | website    | String   | ні          | URL                              |
| Години роботи| workingHours| String  | ні          | Текстом                          |

## 4. Article (стаття / порада)

Колекційний тип для блогу / розділу «Корисна інформація / Поради».

| Поле           | Ключ           | Тип              | Обов’язково | Коментар                               |
|---------------|----------------|------------------|-------------|----------------------------------------|
| Заголовок     | title          | String           | так         | H1 / основний заголовок                |
| Слаг URL      | slug           | UID/Slug         | так         | Для URL `/blog/<article-slug>`     |
| Підзаголовок  | subtitle       | String           | ні          | Короткий вступ                         |
| Прев’ю‑текст  | previewText    | Text             | так         | Тизер для списку статей                |
| Тіло статті   | body           | Rich text        | так         | Основний контент                       |
| Час читання   | readingTimeMin | Integer          | ні          | Орієнтовний час у хвилинах             |
| Теги          | tags           | JSON/Enum (multi)| ні          | Тематичні теги                         |
| SEO title     | seoTitle       | String           | ні          | meta title                             |
| SEO desc      | seoDescription | Text             | ні          | meta description                       |
| Дата публікації| publishedAt   | Datetime         | ні          | Для сортування                         |

## 5. Technology (технологія)

Колекційний тип для опису ключових технологій.

| Поле         | Ключ        | Тип         | Обов’язково | Коментар                           |
|-------------|------------|-------------|-------------|------------------------------------|
| Назва       | name       | String      | так         | Назва технології                   |
| Слаг        | slug       | UID/Slug    | так         | Для внутрішніх посилань            |
| Опис        | description| Rich text   | так         | Опис простою мовою                 |
| Іконка      | icon       | Media       | ні          | Піктограма                         |
| Моделі шин  | tyreModels | Relation(M–N)| ні        | Зв’язок з TyreModel                |

## 6. VehicleFitment (рекомендовані розміри за авто)

Колекційний тип або довідник для підбору шин за маркою/моделлю авто.

| Поле           | Ключ            | Тип           | Обов’язково | Коментар                         |
|---------------|-----------------|---------------|-------------|----------------------------------|
| Марка авто    | make            | String        | так         | Наприклад, Toyota                |
| Модель авто   | model           | String        | так         | Наприклад, Corolla               |
| Тип кузова    | bodyType        | String/Enum   | ні          | sedan, hatchback, suv тощо       |
| Рік з         | yearFrom        | Integer       | так         | Початковий рік                   |
| Рік до        | yearTo          | Integer/null  | ні          | Якщо порожньо — актуальні роки   |
| Рекоменд. розмір| sizes         | Relation (M–N)| так         | Зв’язок з TyreSize               |

## 7. Global SEO / сторінки

Для головної, каталогу, сторінок «Про бренд», «Технології», «Контакти»
можна використати:

- окремий тип **Page** з полем `slug` (`home`, `about`, `technology`, `contacts`)
  і SEO‑полями; або
- конфіг‑тип **Global SEO**, де зберігаються title/description для ключових
  роутів, а тексти блоків редагуються в окремих полях/компонентах.

## 8. Зв’язок з фронтендом

Фронтенд зараз використовує мок‑дані з файлу `src/lib/data.ts`. Під час інтеграції CMS потрібно:

1. Створити в CMS описані вище типи з тими ж полями (або максимально
   близькими за назвою/типом).
2. Налаштувати API (REST/GraphQL) для вибірки:
   - списку моделей шин з розмірами,
   - дилерів,
   - статей,
   - технологій,
   - фітментів авто.
3. У фронтенді замінити звернення до мок‑масивів на fetch до CMS
   (з урахуванням пагінації та фільтрів).

Цей документ можна використати як технічне завдання для налаштування
структури headless CMS під проєкт Bridgestone Україна.
